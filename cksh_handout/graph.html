圖 graph

圖的元素

  • 點
  • 邊

無向邊

連接點u{u}跟v{v}的邊

有向邊

從點u{u}指向v{v}的邊

重邊

從點u{u}跟v{v}有複數個邊

帶權邊

  • 有價值的邊
  • 帶負值為負邊

度/degree

在一個有向圖中 對於一個點：

  • 出度：該點指向其他點的邊的數量 out-degree

  • 入度：其他點指向該點的邊的數量 in-degree

環

環：多個點形成圓圈

自環：有一條有向邊指向自己

連通塊

  • 無向圖中連在一起的稱做連通塊
  • 強連通塊：有向圖中一個點集內任一點皆可走到其他元素

其他

  • 混和圖：同時包含有向邊和無向邊
  • 有向無環圖(DAG)：Literally(之後有更多介紹)
  • 簡單圖：不包含負邊和自環
  • 完全圖：一個簡單圖 且所有點之間皆有一條邊

Q&A

樹

定義/名詞

  • 沒有環的圖
  • 根節點
  • 父/子節點
  • 葉節點
  • 祖先
  • 子樹
  • 深度

根節點

樹的頂端(不一定存在)

父/子節點

*註：根據性別平等法，應稱為長輩/小孩節點

葉節點

沒有子節點的節點們

祖先

父節點以上到根路徑上的所有點

子樹

以其中一個節點作為根的樹

層/深度

距離根節點的距離 根為0

樹直徑

樹的最遠兩個點的路徑

(兩次DFS可完成)

性質

  • N個節點的樹 有N-1條邊
  • 樹沒有環，但加一條邊就會有
  • 樹任兩點都只存在一條簡單路徑

Q&A

DFS

如何遍歷一張圖？

  • 找一個點，找他的一條邊走
  • 然後再找抵達的點的邊走
  • 若走到死路 或沒有尚未走過的路
  • 往回前一個點找其他路走
  • 以遞迴實作

1

往最小數字走

1->2

1->2->6

走回點1

1->2->6->1(x)

因為已到過1 所以回朔至6

1->2->6->5

1->2->6->5->3

1->2->6->5->3->1(x)

1->2->6->5->3->4#

樹DFS

順序：1->2->5->9->3->4->6->7->8

CODE

*如果是dfs一顆樹 通常會順便紀錄父節點

const int maxn = 10;//點的數量
bool vis[maxn];//紀錄該點是否走過
vector<int> edge[maxn];//存邊
void dfs(int n){
    vis[n]=1;//紀錄已走過
    for(int x:edge[n]) if(!vis[x])
        dfs(x,n);
}
int main(){
    dfs(0);//開始
}
